LRU算法实现java版

发表于 2020-02-08

介绍：LRU即为最近最少使用算法，在java中可以使用LinkedHashMap数据结构搞定，LinkedHashMap为双向链表加hashMap的结构，除了hashMap的基本结构，他还多了双向链表的结构，双向链表的作用主要用来保存元素的插入的顺序，换句话来说即双向链表保存了每次插入元素的上一个指向的指针和其下一个指向的指针，即可保证插入的顺序。链表用来表示位置，哈希表用来存储和查找，即可实现LRU算法。


我们可以用一个额外哈希表（例如HashMap）来存放 key-value，这样的话，我们的 get 操作就可以在 O(1) 的时间内寻找到目标节点，并且把 value 返回了。

用了哈希表之后，虽然我们能够在 O(1) 时间内找到目标元素，可是，我们还需要删除该元素，并且把该元素插入到链表头部啊，删除一个元素，我们是需要定位到这个元素的前驱的，然而定位到这个元素的前驱，是需要 O(n) 时间复杂度的。

我们都已经能够在 O(1) 时间复杂度找到要删除的节点了，之所以还得花 O(n) 时间复杂度才能删除，主要是时间是花在了节点前驱的查找上，为了解决这个问题，其实，我们可以把单链表换成双链表，这样的话，我们就可以很好着解决这个问题了，而且，换成双链表之后，你会发现，它要比单链表的操作简单多了。

归并排序及其应用

发表于 2020-02-05 更新于 2020-02-06

归并排序介绍

由分治的思想，将小范围的排好序，再排大范围的

归并排序的应用

应用一： 小和问题

问题：

在一个数组中，每一个数左边比当前数小的数累加起来，叫做这个数组的小和。求一个数组的小和。

例子：

[1,3,4,2,5]

1左边比1小的数，没有；

3左边比3小的数，1；

4左边比4小的数，1、3；

2左边比2小的数，1；

5左边比5小的数，1、3、4、2；

所以小和为1+1+3+1+1+3+4+2=16

类似的问题：逆序对


在一个数组中，左边的数如果比右边的数大，则折两个数构成一个逆序对，请打印所有逆序对。

桶排序及其应用

发表于 2020-02-05 更新于 2020-02-06

桶排的思想

一种不基于比较器的排序，其思想可以参考：

https://www.cnblogs.com/bqwzx/p/11029264.html

桶结构的应用

应用1：求排好序后数组中相邻数最大差值（不用比较排序，并且时间复杂度为o(N)）

题目：

给定一个数组，求如果排序之后，相邻两数的最大差值，要求时 间复杂度O(N)，且要求不能用非基于比较的排序。

思路：

由上面可知，不用比较的排序，那么只有桶排序，根据桶排序的思想，需要把数组的数装到桶中，这里需要注意，这里的桶是装一个范围的数，不只是一个数（例如有10个桶，第0个桶装数的范围为0~10，以此类推）假如有N个桶，那么我们需要N+1个桶（为什么呢？因为要有一个空桶，其他桶都是有装数的，这样设置的目的是为了让最大的差值来自于两桶之间，而非桶内，即如果只设置N个桶，那么有可能差值最大值来自于桶内）。


接下来我们需要把数装到桶中，每个桶设置3个量，一个为布尔值表示是否装有数，一个整形max值表示桶中最大值，一个整形min值表示桶中最小值，注意，其他值不需要装入桶中，这样子拿两个非空桶中后一个桶的最大值减前一个桶的最小值即可得到两个非空桶的最大差值。（这三个量可以用3个数组表示，数组的大小为桶的个数）


接下来再设置一个全局的整形，用来返回数组最大差值，通过遍历数组依次得到两个非空桶的差值，再和全局的最大差值比较得到其中较大者，最后得到结果。（注意：空桶两边的桶的差值并不一定是最大差值，其只是保证最大差值来自桶间）

堆及其应用

发表于 2020-02-05

一、堆的概念及其属性

1.1概念

堆就是用数组实现的二叉树（完全二叉树），所有它没有使用父指针或者子指针。堆根据“堆属性”来排序，“堆属性”决定了树中节点的位置。

1.2堆的属性

  • 堆具有完全二叉树的属性：当父子节点为在数组的下标为i，则其左子节点为2i+1，右子节点为2i+2；当已知子节点为i，其父子节点为(i-1)/2；
  • 堆只有大根堆和小跟堆，两个必须满足其中一个；在最大堆中，父节点的值比每一个子节点的值都要大。在最小堆中，父节点的值比每一个子节点的值都要小。

二、堆的操作（以大根堆为例）

2.1插入一个元素到堆尾后再次堆化（建立一个堆的操作）

2.2删除一个元素后再次堆化

删除堆中元素的操作：1.将要删除的位于堆顶元素与堆中最后一个元素交换；2.堆的大小减一；3.再次堆化（注意与插入的堆化不同，这是一个下浮的过程）

三、堆的应用（利用堆结构）

说明：因为堆结构是一种特别适合用来找第k小/大的特殊的数据结构。

3.1堆排序

已知堆中的最大值为堆顶，可以将堆顶剔除出堆，然后再次堆化，即可将数组有序化，其时间复杂度为o(NlogN);

3.2应用2：求数据流的中位数

在一个不断吐出数据的数据流中，如何实时的得到吐出的数据的中位数

思路：用堆结构解决

建立一个大根堆和一个小根堆

具体的步骤为：

  • 当大小根堆都为空时，先插入大根堆
  • 第N个数插入时，先判断第N个数是否大于大根堆堆顶，若小于则插入大根堆（要插入堆尾，再堆化），否则插入小根堆。
  • 判断大根堆大小与小根堆大小之差是否大于1，若大于，需要从较大者堆中弹出堆顶，加入到令一个较小的根堆（此时大小根堆都维持相等的个数，两个根堆的堆顶之和除2即为中位数）

滑动窗口算法(双指针算法的变形)

发表于 2020-02-01 更新于 2020-02-03

参考引用： https://www.zhihu.com/topic/20746237/intro

http://www.imooc.com/article/285643

一、简介

滑动窗口算法可以用以解决数组/字符串的子元素问题，它可以将嵌套的循环问题，转换为单循环问题，降低时间复杂度。

解题思路：

  • 确定窗口
  • 遍历字符串

二、示例

示例1：求长度k的连续子串最大总和

问题引述：


给定一个整数数组，计算长度为 ‘k’ 的连续子数组的最大总和。
输入：arr [] = {100,200,300,400}

k = 2


输出：700

解释：300 + 400 = 700

示例2：字符串子串问题

给定一个字符串 S 和一个字符串 T，请在 S 中找出包含 T 所有字母的最小子串。(minimum-window-substring)

输入: S = “ADOBECODEBANC”, T = “ABC”

输出: “BANC”

思路:

我们仍然可以使用滑动窗口算法，只不过需要换一个思路。
既然是找最小的窗口，我们先定义一个最小的窗口，也就是长度为 0 的窗口。



我们比较一下当前窗口在的位置的字母，是否是 T 中的一个字母。

很明显， A 是 ABC 中的一个字母，也就是 T 所有字母的最小子串 可能包含当前位置的 S 的值。

如果包含，我们开始扩大窗口，直到扩大后的窗口能够包含 T 所有字母。





假设题目是 在 S 中找出包含 T 所有字母的第一个子串，我们就已经解决问题了，但是题目是找到最小的子串，就会存在一些问题。

  • 当前窗口内可能包含了一个更小的能满足题目的窗口
  • 窗口没有滑动到的位置有可能包含了一个更小的能满足题目的窗口


    为了解决可能出现的问题，当我们找到第一个满足的窗口后，就从左开始缩小窗口。

  • 如果缩小后的窗口仍满足包含 T 所有字母的要求，则当前窗口可能是最小能满足题目的窗口，储存下来之后，继续从左开始缩小窗口。
  • 如果缩小后的窗口不能满足包含 T 所有字母的要求，则缩小窗口停止，从右边开始扩大窗口。



    不断重复上面的步骤，直到窗口滑动到最右边，且找不到合适的窗口为止。最小满足的窗口就是我们要找的 S 中包含 T 所有字母的最小子串。

    示例3：求不重复的最长子串

    给定一个字符串，请你找出其中不含有重复字符的 最长子串 的长度。(longest-substring-without-repeating-characters)

    输入: “abcabcbb”

    输出: 3

    解释: 因为无重复字符的最长子串是 “abc”，所以其长度为 3。


思路：和示例 2 相似，我们不断的扩大/缩小窗口，把无重复字母的窗口大小保存下来，直到窗口滑动结束，就找到了不含有重复字符的 最长子串 的长度。

示例3：求滑动窗口最大值

问题：

给定一个数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口 k 内的数字。滑动窗口每次只向右移动一位。
返回滑动窗口最大值。

示例:
输入: nums = [1,3,-1,-3,5,3,6,7], 和 k = 3

输出: [3,3,5,5,6,7]

解释:

滑动窗口的位置    最大值

————— —–

[1 3 -1] -3 5 3 6 7    3

1 [3 -1 -3] 5 3 6 7    3

1 3 [-1 -3 5] 3 6 7    5

1 3 -1 [-3 5 3] 6 7    5

1 3 -1 -3 [5 3 6] 7    6

1 3 -1 -3 5 [3 6 7]    7

题目解析

利用一个 双端队列，在队列中存储元素在数组中的位置， 并且维持队列的严格递减,，也就说维持队首元素是最大的，当遍历到一个新元素时, 如果队列里有比当前元素小的，就将其移除队列，以保证队列的递减。当队列元素位置之差大于 k，就将队首元素移除。


补充：什么是双端队列（Dqueue）

Deque 的含义是 “double ended queue”，即双端队列，它具有队列和栈的性质的数据结构。顾名思义，它是一种前端与后端都支持插入和删除操作的队列。

Deque 继承自 Queue（队列），它的直接实现有 ArrayDeque、LinkedList 等。

双指针算法

发表于 2020-01-09 更新于 2020-02-05

相向指针

两个相向的指针，在算法的一开始，两根指针分别位于数组/字符串的两端，并相向行走。

应用1：翻转字符串

应用2：两数之和

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。
你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

思路：首先我们对数组进行排序。

用两个指针(L, R)从左右开始：

如果numbers[L] + numbers[R] == target, 说明找到，返回对应的数。
如果numbers[L] + numbers[R] < target, 此时L指针右移，只有这样才可能让和更大。
反之使R左移。

应用3：判断回文串

另外一个双指针的经典练习题，就是回文串的判断问题。给一个字符串，判断这个字符串是不是回文串。

同向双指针

即两个指针都是同向的，但是一个指针快，一个指针慢，并且快的指针会遍历整个数组或者链表。

应用一：数组去重问题

首先将数组排序，这样那些重复的整数就会被挤到一起。然后用两根指针，一根走得快一些遍历整个数组，另一根指针一直指向当前不重复的部分的最后一个数。当快指针发现和慢指针指向的数不同之后，先对慢指针++，然后把快指针指向的数赋予给慢指针的指向的最后一个的位置。

应用2：滑动窗口问题

问题引述: 这个问题并没有什么难度，但是如果你过于暴力的用户O(n * k) 的算法去做是并不合适的。比如当前的 window 是 |1,2|,3,4。那么当 window 从左往右移动到 1,|2,3|,4 的时候，整个 window 内的整数和是增加了3，减少了1。因此只需要模拟整个窗口在滑动的过程中，整数一进一出的变化即可。这就是滑动窗口问题。
时间复杂度O(n),空间复杂度O(n-k+1)
